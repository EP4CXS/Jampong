# Voice Control Integration Plan for Super Dash

## Overview
Integrate voice control into the Flutter + Flame Super Dash project using the speech_to_text package. The bird/player should jump when user says "jump", "fly", "up", or "go".

## Files to Modify

### 1. pubspec.yaml
- Add speech_to_text dependency

### 2. Android Manifest
- android/app/src/main/AndroidManifest.xml
- Add RECORD_AUDIO permission

### 3. iOS Info.plist
- ios/Runner/Info.plist
- Add NSMicrophoneUsageDescription and NSSpeechRecognitionUsageDescription

### 4. New Service
- lib/services/voice_control_service.dart
- Create VoiceControlService class to handle speech recognition

### 5. Game View Integration
- lib/game/view/game_view.dart
- Initialize and dispose VoiceControlService
- Add UI toggle button and status indicator

### 6. Optional: Connection to Game
- Connect voice commands to existing input system via addInputListener

## Implementation Details

### VoiceControlService Responsibilities
- Initialize speech_to_text
- Start/stop listening
- Listen for commands: "jump", "fly", "up", "go"
- Implement debounce to prevent multiple jumps per command
- Callback mechanism to trigger jump
- Handle errors and permissions gracefully

### Integration Points
- Register voice service as input listener similar to PlayerControllerBehavior
- OR directly call jump logic when voice command detected
- Prefer reusing existing input listener system for consistency

### UI Elements
- Toggle button: "Voice: ON"/"Voice: OFF" (top-right)
- Status text showing: "Voice ready", "Listening...", etc.

### Commands Supported
- jump
- fly
- up
- go

### Testing Steps
1. flutter pub get
2. flutter run (on Android/iOS device or emulator with microphone)
3. Say voice commands during gameplay
4. Verify jump action occurs
5. Verify existing tap/keyboard controls still work

## Implementation Order
1. Update pubspec.yaml
2. Add platform permissions
3. Create VoiceControlService
4. Integrate with game view
5. Add UI controls
6. Test and verify
